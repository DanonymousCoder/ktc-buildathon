# FlowTrakka Local Backend

This is a tiny Node backend for the shared FlowTrakka leaderboard and landing-page waitlist. It stores aggregate opt-in stats and submitted waitlist emails, but not document titles, document URLs, contents, or raw session history.

## Run Locally

```bash
npm run leaderboard:server
```

The server listens on `http://localhost:8787` by default.

## Endpoints

- `GET /health`
- `GET /api/leaderboard`
- `POST /api/leaderboard/entries`
- `POST /api/waitlist`

`POST /api/leaderboard/entries` expects the aggregate payload generated by the extension. The server rejects payloads that do not explicitly mark document titles, document URLs, and raw session history as excluded.

`POST /api/waitlist` expects `{ "email": "reader@example.com", "source": "hero" }`. It validates and normalizes the email, then stores one record per address.

## Production

Deploy this folder to a Node host, set `PORT` if your host requires it, then update the extension's Leaderboard API URL setting to your HTTPS backend URL.

Leaderboard and waitlist data are saved to `leaderboard-server/data/leaderboard.json` locally. In production, use the Cloudflare Worker and D1 backend rather than the local JSON store.
